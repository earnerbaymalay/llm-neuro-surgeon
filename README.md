## The problem

If you use more than one AI coding tool, you maintain the same knowledge many
times over:

- Skills and `CLAUDE.md` for **Claude Code**
- `GEMINI.md` for **Gemini CLI**
- `AGENTS.md` for **Codex / OpenCode / Zed**
- `.cursor/rules` for **Cursor**, `.clinerules` for **Cline / Roo**
- `CONVENTIONS.md` for **Aider**, `copilot-instructions.md` for **Copilot**
- …plus a separate **MCP server config** for each of them.

Formats drift. Skills go stale in some tools but not others. And there's no
single place that shows what any given model actually "knows."

## The solution

LLM Neurosurgeon performs **"neurosurgery" on your machine**:

1. **Scans** for every AI tool's config.
2. **Imports** their skills / agents / rules / memory / prompts / MCP servers,
   losslessly, into **one canonical directory — the Brain** (`~/AIBrain` by
   default, git-backed).
3. **Projects** that Brain back out to every tool — via symlink where the tool
   tolerates it, or a generated `GENERATED BY … — edit in the Brain` file where
   it doesn't.
4. **Keeps it in sync** with a filesystem watcher + scheduled sweeps. Edit once,
   in the Brain, and every model stays equally skilled — forever.

Every sync is a git commit, so you get a **Time Machine**: browse, diff, and
roll back your entire AI configuration history.

## ✨ Feature pillars

| Pillar | What it does |
|---|---|
| 🧠 **The Brain** | One central, human-readable, git-backed directory that is the single source of truth. |
| 📥 **Universal Import** | Adapters that losslessly ingest configs from 12 tools (list below). |
| 📤 **Projection Engine** | Per-tool output: symlink where tolerated, generated file otherwise; first-class `AGENTS.md` target. Windows junction/hardlink/copy fallbacks. |
| 🔄 **Auto-Sync Daemon** | Debounced watcher + scheduled sweeps, bidirectional, three-way merge with a GUI conflict queue. Every sync = a git commit. |
| 🛒 **Marketplace Import** | One-click import of skills/agents from `anthropics/skills`, awesome-lists, Gemini extensions, or any git URL — with provenance, license, and diff preview. |
| 🔌 **MCP Hub** | Browse/search/install MCP servers, project the config to every tool, health-check each server, store secrets in the OS keychain. |
| 🩺 **Vitals & Doctor** | Coverage matrix (tool × capability), drift/broken-link detection, and one-click diagnostic fixes. |
| 🛡️ **Safety by design** | Dry-run default, snapshot before any destructive op, checksum after copy, untrusted-import flagging, no telemetry, offline except explicit fetches. |

**Non-goals (v1):** running models, editing chat histories, cloud sync, mobile.

## 🗂️ The canonical Brain layout

```text
AIBrain/
  skills/<slug>/        SKILL.md + skill.yaml (id, version, triggers, targets, source, sha256)
  agents/<slug>.md      canonical agent defs (frontmatter: tools, model hints, targets)
  rules/                global.md + scoped/<glob>.md   (single source for all *.md rule files)
  memory/               MEMORY.md + topic files
  prompts/              reusable commands / templates
  mcp/servers/<id>.yaml (transport, command/url, env-placeholders, targets, health)
  .brain/               mappings.json (source↔canonical↔projection + hashes), state.json, locks, logs/, backups/
  .git/                 full history (the "Time Machine")
```

## 🏛️ Architecture

A Rust core does the filesystem-heavy work and is shared by both the desktop
GUI (via Tauri commands) and the CLI (via a library crate).

```text
┌─────────────────────┐        ┌────────────────────┐
│  apps/desktop (GUI) │        │   apps/cli (clap)  │
│  Tauri + React/TS   │        │  scan import ...   │
└──────────┬──────────┘        └─────────┬──────────┘
           │  Tauri commands             │  lib API
           └──────────────┬──────────────┘
                          ▼
              ┌───────────────────────┐
              │   packages/core (Rust)│
              │  scanner · canonical  │
              │  model · adapters ·   │
              │  projector · sync     │
              └───────────┬───────────┘
                          ▼
            filesystem · git · OS keychain
```

**Core sync loop:**

```text
on watch_event | tick:
  for each mapping: h_src=hash(source); h_brain=hash(canonical)
    src changed only   -> import(source) -> project(all other targets)
    brain changed only -> project(all targets)
    both changed       -> three_way_merge(base=last_snapshot)
                          disjoint-md ? auto : conflict_queue.push
  git.commit("sync: <summary>"); update mappings.json hashes
```

## 🤖 How this project is built: the RALPH loop

This repo is built by an autonomous agent following a **file-state-over-chat**
discipline — all memory lives in the repo, not in a conversation:

- [`MASTER_PROMPT.md`](MASTER_PROMPT.md) — the self-executing spec (8 phases,
  5 human gates).
- [`PLAN.md`](PLAN.md) — every task as `- [ ] T<phase>.<n> … :: verify: <check>`.
- [`PROGRESS.md`](PROGRESS.md) — append-only log, ≤3 lines per task.
- [`DECISIONS.md`](DECISIONS.md) — one line + rationale per technical decision.
- [`RALPH_PROMPT.md`](RALPH_PROMPT.md) + [`ralph.sh`](ralph.sh) — the inner
  loop: read plan → pick the single highest-priority unblocked task →
  implement → run its `verify:` → commit → tick the box → stop.

Each iteration starts with fresh context (the real token saver), and the loop
**halts at every `[GATE]`** for human review.

## 🛡️ Security & privacy

No telemetry. Fully offline except explicit marketplace/registry fetches.
Imports never follow symlinks outside scanned roots; imported skills are treated
as untrusted prompt/code (source + checksum shown, disabled by default). Secrets
live in the OS keychain and are written to tool configs as env placeholders only.
