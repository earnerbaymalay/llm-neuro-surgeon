## 💡 The Problem

If you use more than one AI coding tool (such as Claude Code, Cursor, Gemini CLI, Aider, Windsurf, GitHub Copilot, or Zed), you maintain the same knowledge many times over:

- Skills, prompts, and `CLAUDE.md` for **Claude Code**
- `GEMINI.md` and `.gemini/settings.json` for **Gemini CLI**
- `AGENTS.md` for **OpenCode / Codex / Zed**
- `.cursorrules` and `.cursor/rules/*.mdc` for **Cursor**
- `.clinerules` and `.roomodes` for **Cline / Roo Code**
- `CONVENTIONS.md` for **Aider**, `copilot-instructions.md` for **GitHub Copilot**
- …plus separate **MCP server configs** for each tool.

Formats drift over time, skills go stale in some tools while updated in others, and there is no single place that shows what any given model actually "knows."

---

## ⚡ The Solution

LLM Neurosurgeon performs **"neurosurgery" on your machine**:

1. **Scans** for every AI tool's configuration files across your workspace and home directory.
2. **Imports** their skills, agents, rules, memory, prompts, and MCP servers losslessly into **one canonical directory — the Brain** (`~/AIBrain` by default, backed by Git).
3. **Projects** that Brain back out to every tool — using symlinks where tolerated or generated files stamped with `<!-- GENERATED BY LLM NEUROSURGEON — edit in the Brain -->` where required.
4. **Keeps it in sync** using a debounced filesystem watcher and OS background schedulers (`launchd`/`systemd`/Task Scheduler). Edit once in the Brain, and every model stays equally skilled — forever.

Every sync operation is a Git commit, providing a complete **Time Machine**: browse history, diff changes, and roll back your entire AI configuration at any time.

## ✨ Feature Pillars

| Pillar | What it does |
|---|---|
| 🧠 **The Brain** | One central, human-readable, git-backed directory (`~/AIBrain`) acting as the single source of truth. |
| 📥 **Universal Import** | Built-in adapters that losslessly ingest configs from 12 AI coding tools. |
| 📤 **Projection Engine** | Per-tool output engine: symlinks where tolerated, generated files where required, merged JSON/TOML configurations, and first-class `AGENTS.md` support. |
| 🔄 **Auto-Sync Daemon** | Debounced filesystem watcher + background OS schedulers (`launchd`, `systemd`, Task Scheduler). 3-way merge with conflict queue. Every sync = a Git commit. |
| 🛒 **Marketplace Import** | Ingest skills and agents from `anthropics/skills` or any Git repository — with license cards, SHA-256 provenance, and executable script warnings (disabled by default). |
| 🔌 **MCP Hub** | Browse/search MCP servers, project configurations to every tool, execute stdio/remote health-check handshakes, and store API keys in the OS Keychain using `${VAR}` placeholders. |
| 🩺 **Vitals & Doctor** | Health matrix (tool × capability), drift/broken-link detection, and `neurosurgeon doctor --fix` for one-click diagnostic repairs. |
| 🛡️ **Safety by Design** | Dry-run defaults, Git snapshot before destructive ops, checksum verification, local-first execution, and zero telemetry. |

## 🗂️ Canonical Brain Layout

```text
AIBrain/
├── skills/<slug>/        # SKILL.md + skill.yaml (id, version, triggers, targets, source, sha256)
├── agents/<slug>.md      # Canonical agent definitions (frontmatter: tools, model hints, targets)
├── rules/                # global.md + scoped/<glob>.md (single source for all rules)
├── memory/               # MEMORY.md + topic files
├── prompts/              # Reusable commands / templates
├── mcp/servers/<id>.yaml # Transport, command/url, env-placeholders, targets, health
├── .brain/               # mappings.json (source ↔ canonical ↔ projection + hashes), state.json, locks
└── .git/                 # Git history (the "Time Machine")
```

## 🚀 Quickstart & Usage

### Prerequisites
- **Rust**: 1.75+
- **Node.js**: 20+ (`pnpm` workspace)
- **Git**

### CLI Commands

```bash
# Scan current directory for installed AI tools
cargo run -p neurosurgeon -- scan

# Run a dry-run import preview (nothing written to disk)
cargo run -p neurosurgeon -- import --dry-run

# Run Doctor diagnostics to detect configuration drift or broken symlinks
cargo run -p neurosurgeon -- doctor

# Automatically repair fixable Doctor diagnoses
cargo run -p neurosurgeon -- doctor --fix

# Record a manual Git snapshot of the Brain
cargo run -p neurosurgeon -- snapshot "Updated global rules"

# Roll back Brain state to a prior snapshot
cargo run -p neurosurgeon -- rollback <commit-hash>
```

### Desktop App Setup

```bash
# Build desktop app frontend & run Tauri development window
cd apps/desktop
pnpm install
pnpm tauri dev
```

## 🛡️ Security & Privacy

- **Zero Telemetry**: No tracking, usage analytics, or remote data collection.
- **Local First**: All scanning, importing, projecting, and sync logic runs offline on your machine.
- **OS Keychain Secrets**: API keys are stored in your native OS Keychain and written to configuration files as `${VAR}` environment placeholders only.
- **Path Traversal Defense**: All path joins pass through `safe_join()` to block symlink escape and relative path traversal attacks.
